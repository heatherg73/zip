Step-by-step visual guide to setting up the Wagtail CMS project

Wagtail CMS — setup guide
From zip file to running site in 10 steps  ·  Python + Node.js required
Prerequisites
1
Install Python 3.10+
Download from python.org/downloads and run the installer.
Windows users: tick "Add Python to PATH" before clicking Install — don't skip this.
python --version  # should print Python 3.10 or higher
2
Install Node.js 18+
Required for Tailwind CSS. Download from nodejs.org — pick the LTS version.
node --version  # should print v18 or higher
3
Install VS Code
Download from code.visualstudio.com. Then install these extensions from the Extensions panel (Ctrl+Shift+X):
Python
IntelliSense & debugger
Tailwind CSS IntelliSense
Autocomplete in templates
Django
Template syntax highlighting
SQLite Viewer
Browse the database
Project setup
4
Unzip and open in VS Code
Unzip wagtail_cms_project.zip, then open VS Code → File → Open Folder → select the wagtail_cms folder.
# Or from terminal:
cd ~/Downloads && unzip wagtail_cms_project.zip
code wagtail_cms
5
Create a virtual environment
Open the VS Code terminal with Ctrl+` and run:
Windows
python -m venv venv
venv\Scripts\activate
macOS / Linux
python3 -m venv venv
source venv/bin/activate
You'll see (venv) at the start of your terminal prompt when it's active. VS Code may also prompt you to select this as the Python interpreter — click Yes.
6
Install Python & Node dependencies
# Python packages
pip install -r requirements.txt

# Tailwind / Node packages
cd theme/static_src
npm install
cd ../..
7
Set up the database
# Create all database tables
python manage.py migrate

# Create your admin login
python manage.py createsuperuser
Pick a password like Admin1234! — must be 8+ characters, not all numbers, not too simple.
8
Build Tailwind CSS
cd theme/static_src
npm run build
cd ../..
Running the project
9
Start two terminals side by side
Split the terminal with the + icon (or Ctrl+Shift+5). Run one command in each:
Terminal 1 — Django
python manage.py runserver
Terminal 2 — Tailwind
cd theme/static_src
npm run start
10
Open in your browser
Sitehttp://127.0.0.1:8000/
Adminhttp://127.0.0.1:8000/cms/
Sitemaphttp://127.0.0.1:8000/sitemap.xml
Translationshttp://127.0.0.1:8000/rosetta/
Log in to the admin with the username and password you created in step 7.
Fix email (optional)
Enable real email sending
By default emails print to the terminal. To actually send them, open mysite/settings.py and replace the email section:
# Replace this:
EMAIL_BACKEND = 'django.core.mail.backends.console.EmailBackend'

# With this (Gmail example):
EMAIL_BACKEND = 'django.core.mail.backends.smtp.EmailBackend'
EMAIL_HOST = 'smtp.gmail.com'
EMAIL_PORT = 587
EMAIL_USE_TLS = True
EMAIL_HOST_USER = 'you@gmail.com'
EMAIL_HOST_PASSWORD = 'your-app-password'
DEFAULT_FROM_EMAIL = 'you@gmail.com'
Gmail requires an App Password — not your regular password. Enable 2FA on your Google account first, then generate one at myaccount.google.com/apppasswords.
